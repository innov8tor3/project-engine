Project summary

Build a public-repository security audit tool. The front end is on GitHub Pages and the backend is a Python service that scans a repository path such as /docs recursively.

The audit classifies files as red (unsecure), green (secured), or grey (unnecessary / out of scope).

Security model

  • This tool is for public repositories only.
  • Private repository scanning should be discussed separately with platform admins.
  • No user token field should be used in the public version.
  • The backend may use its own server-side token later if needed, but the browser should not.

Audit rules

The backend should scan files recursively and flag these five patterns:

  1. innerHTML
  2. Inline event handlers such as onclick=
  3. target="_blank" links missing rel="noopener noreferrer"
  4. unsafe-inline in CSP-related files
  5. Client-side GitHub token handling or Authorization: token

Known deployment target

The backend is intended to be deployed on Render as a Python web service.
