🔒 Securing project-engine

A beginner’s real journey with GitHub security • April 2026

Why This Took So Much Time & Effort

This was not quick or easy.

As a complete beginner repo owner, setting up even basic GitHub security involved many small confusing steps, hidden settings, UI glitches (like the “Configure” message that never went away), and moments of frustration. It required patient AI guidance, repeated checking, and learning concepts like Pull Requests, branch protection, and Dependabot for the first time.

It took real time, persistence, and encouragement. Many repo owners give up halfway. That’s why documenting it matters.

Why This Work Is Worth It

Even for a mostly static HTML/CSS site like project-engine (a project to feed other projects by tokenising assets), basic security protects:

  • Your code and assets from accidental damage
  • Users and contributors from hidden vulnerabilities
  • Your reputation as a responsible maintainer
  • Future scalability — when the site grows, the foundation is already solid

GitHub now shows a clean green Security overview. Future changes are safer. You and your users benefit.

What We Achieved – Your Current Setup

  • ✅ .github folder with dependabot.yml, copilot-instructions.md, and security workflow
  • ✅ Dependabot Version Updates running (history visible, even if UI says “Configure”)
  • ✅ SECURITY.md policy file (visible on Security tab)
  • ✅ Basic branch protection on main (requires PRs + passing checks)
  • ✅ CodeQL / Code Scanning enabled and tested
  • ✅ Secret scanning and Dependabot alerts active (all green in Security overview)

Your repo is now in a much stronger position than most small static sites.

Future Checklist – Safe Workflow for Any Update

Print or bookmark this. Follow it every time you want to change something.

  1. Never edit directly on main (branch protection now prevents risky direct pushes)
  2. Make your change on GitHub → create a new branch when prompted
  3. Create a Pull Request (PR) to main
  4. Wait for green checks (CodeQL, security scan, etc.)
  5. Review the changes yourself
  6. Use “Squash and merge” for small changes
  7. Delete the branch after merging
  8. Check the Security tab regularly for new alerts
  9. When Dependabot opens a PR → review it, merge safe ones (they keep your workflows updated)

Tip: If anything feels confusing again, come back to this page or ask your AI helper step-by-step.

A Note to Other Beginners

If you’re following along: Yes, this process really does take time and patience. You will hit confusing screens, temporary glitches, and moments where you feel lost. That’s normal.

Use AI tools generously. Ask for click-by-click instructions. Take breaks. Celebrate small wins (like seeing your first green checkmarks or successfully merging your first PR).

The effort is worth it — for you as the repo owner, and for everyone who uses or builds on your project.
